# Demo

You've built a real agent. Last step: make it presentable, then show it on stage.

## Step 1 — Quick polish

Send your agent 3–4 messages and read the replies. If something feels off, tell your AI in your own words what to change — *"too formal," "keep replies shorter," "more sarcasm," "don't use that emoji"* — and let it update the personality. Do one more pass if needed. (No restart — it reads the personality fresh every reply.)

:::py
💻 **RUN**
```bash
python core.py "hey, what's up"
```
:::
:::js
💻 **RUN**
```bash
node core.js "hey, what's up"
```
:::

## Step 2 — Name your agent

Your agent needs an identity for the demo. If you named it in ideation, use that name everywhere — in its personality and on Slack. If not, name it now: tell your AI *"my agent's name is ___"* and have it update the personality (and the Slack display name).

## Step 3 — The demo (on stage)

Keep it honest and simple — about a minute:

1. Introduce yourself.
2. Introduce your agent by name.
3. One line: what it does, and who it's for.
4. Show it live — send a real message and let everyone see the real reply. On Slack? Show it replying in Slack.

> 💬 Show the real thing — don't read a script or fake it. A live reply that sounds like *you* is the whole point.

## Step 4 — Before you go up

- Have it already running (terminal / server / Slack open).
- Have 2–3 messages ready to send.
- If something breaks, stay calm and talk through what you built — everyone here knows it's a 3-hour build.

**You built a real personal AI agent. Go show it off.** 🚀
